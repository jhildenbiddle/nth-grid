Nth-Grid is a lightweight and semantic CSS grid system available for Less, PostCSS, Sass and Stylus. The purpose of Nth-Grid is to provide a versatile grid-based layout system that balances simplicity and flexibility while addressing commonly criticized aspects of other grid systems.

Why Nth-Grid?

There are many CSS frameworks and grid systems to choose from, each with their own features, build requirements and grid syntax. Nth-Grid stands out in this crowded field by providing all of the features expected of a modern CSS grid system, introducing new features that are unique to Nth-Grid, and simplifying grid concepts and implementation.

  • Custom grids without limitations

    Nth-Grid provides a level of flexibility that no other grid system offers: multiple custom grids on the same page, fixed and fluid columns in the same grid, fluid grids with fixed gutters, multi-row grids, nested grids, column reordering, equal height columns, vertically-aligned columns, RTL support, and on-the-fly grid changes for your responsive layouts.

  • Support for modern and legacy browsers

    Most grid systems have abandoned support for legacy browsers, leaving projects that require legacy support with a limited selection of grid systems to choose from. Nth-Grid supports a variety of grid layouts that are compatible with legacy browsers and enhanced functionality for modern browsers that support the latest CSS features.

  • Semantic grids without CSS classes

    Other grid systems require adding CSS classes to your HTML markup for containers, rows, columns, reordering, nesting, clearing floats, responsive breakpoints, etc. The end result is markup littered with grid-specific CSS classes, adding unnecessary bloat and making changes a time-consuming challenge. Nth-Grid keeps your markup clean and the CSS for your grid-based layouts where it belongs: in your CSS.

    <!-- My 3 column grid -->
<main>
    <div>First column</div>
    <div>Second column</div>
    <div>Third column</div>
</main>
    HTML

  • Native support for Less, Sass, Stylus & PostCSS

    The flexibility to work with the languages, frameworks, applications and command-line tools you are already familiar with provides the fastest integration with the smallest learning curve possible. This versatility also allows for adoption of a single grid system across multiple projects: create grid layouts using Less.js in the browser, Sass with Ruby, Stylus with CodeKit, or PostCSS with Node.js and Gulp. The choice is yours.

    // My 3 column grid (Less)
main {
    .nth-grid(
        @columns: 3,
        @gutter: 1%
    );
}
    /* My 3 column grid (PostCSS) */
main {
    nth-grid(
        columns: 3;
        gutter: 1%;
    );
}
    // My 3 column grid (Sass)
main {
    @include nth-grid(
        $columns: 3,
        $gutter: 1%
    );
}
    // My 3 column grid (Stylus)
main
    .nth-grid(
        $columns: 3
        $gutter: 1%
    )
